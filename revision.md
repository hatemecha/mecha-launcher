# Revision del proyecto `mecha-launcher`

## Summary
Proyecto Tauri + Svelte + Rust con buena base funcional en la parte del launcher, pero con varios problemas de coherencia interna, resiliencia de producto, seguridad del webview, deuda estructural y arrastre de iteraciones anteriores.

Verificacion realizada:
- `npx tsc --noEmit`: OK.
- `cargo test --target-dir target-review`: OK.
- `npm run build` y `npx vite build --configLoader native`: no verificables en este sandbox porque Vite falla con `spawn EPERM` antes de resolver el proyecto; no lo tomo como defecto confirmado del repo.

## Critical
- **La inicializacion del launcher depende de que cargue OptiFine.** Impacto: si falla `listOptifineInstallOptions()` por red, timeout o cambio de HTML, el `catch` externo aborta tambien la carga de dependencias, la deteccion de `.minecraft` y el refresco de versiones locales, dejando inutilizable incluso el modo local. Evidencia: `src/App.svelte:995-1017`. Correccion: separar la inicializacion local de los catalogos remotos y capturar cada carga de red por separado.
- **La estrategia de Java se contradice a si misma y puede romper versiones soportadas.** Impacto: el panel recomienda e instala siempre Java 21 (`src-tauri/src/commands.rs:146-259`, `364-389`), pero el launcher exige coincidencia exacta con `majorVersion` (`src-tauri/src/launcher/java.rs:18-36`). Resultado: el flujo "instalar Java automaticamente" no garantiza que una version compatible arranque y puede dejar afuera versiones que requieren 8/16/17. Correccion: resolver Java por version seleccionada, instalar el major correcto o aceptar una politica explicitamente definida y consistente entre UI y backend.

## High
- **La localizacion esta fragmentada y no hay una unica fuente de verdad.** Impacto: la app muestra mezcla de espanol e ingles en UI, estados, logs y errores; ademas impide una traduccion completa. Evidencia: frontend con strings hardcodeados en `src/App.svelte:284,292,1202,1207,1363-1377`; backend con mensajes en ingles en `src-tauri/src/launcher/process.rs:46,83,106` y `src-tauri/src/commands.rs:773`, y en espanol en `src-tauri/src/launcher/install.rs:297-298,398-470,497-564` y `src-tauri/src/launcher/mod.rs:199-212`. Correccion: mover todos los textos de UX a una capa comun y tratar los errores tecnicos del backend como codigos/estructuras, no como copy final.
- **`App.svelte` concentra demasiadas responsabilidades.** Impacto: el componente mezcla estado global, storage, i18n, bootstrap, networking, catalogo, instalacion, borrado, play flow y markup en un solo archivo de 1591 lineas, lo que eleva el costo de cambio y hace dificil testear o razonar regresiones. Evidencia: `src/App.svelte` y handlers repartidos en `266,299,491,535,800,859,915,973,1445-1560`. Correccion: extraer stores/servicios para bootstrap, catalogo, dependencias y acciones del launcher, y dividir la UI en componentes por seccion.
- **La superficie del webview esta demasiado abierta para una app de escritorio.** Impacto: `csp` esta deshabilitado, las fuentes se bajan desde Google en runtime y encima se usa `@html`; hoy no hay una explotacion obvia, pero la combinacion aumenta innecesariamente la superficie de inyeccion y agrega dependencia de red/privacy para renderizar. Evidencia: `src-tauri/tauri.conf.json:14`, `src/app.css:1`, `src/App.svelte:1394,1469`. Correccion: definir CSP explicita, empaquetar fuentes locales y eliminar `@html` donde no sea estrictamente necesario.
- **El parser de OptiFine asume que el primer match del HTML es la build mas nueva y la etiqueta como "Ultima" sin comprobarlo.** Impacto: si cambia el orden de la pagina o aparece otra variante primero, se puede ofrecer una build incorrecta mientras la UI afirma que es la mas reciente. Evidencia: `src-tauri/src/launcher/install.rs:268-298`, especialmente `279` y `297-298`. Correccion: parsear version/edicion de forma deterministica, ordenar por build real y degradar con cache o error controlado si el scraping deja de ser confiable.
- **No hay cobertura automatizada del frontend ni de los flujos UI criticos.** Impacto: toda la complejidad de seleccion, filtros, borrado, instalacion, dependencias, i18n y bootstrap queda sin red de seguridad. Evidencia: `package.json:6-19` solo define `dev/build/preview/tauri`; no hay scripts ni dependencias de pruebas frontend. Correccion: agregar al menos pruebas de componentes/estado y una capa de smoke e2e para los flujos principales del launcher.

## Medium
- **Las versiones locales desconocidas/custom se etiquetan como `vanilla`.** Impacto: la UI presenta informacion falsa para instalaciones locales que no vienen del catalogo remoto y puede inducir acciones equivocadas sobre mods/custom builds. Evidencia: `src/App.svelte:950-964` comenta "offline/unknown/custom" pero crea items con `kind: "vanilla"` y `subtitle: "Local"`. Correccion: introducir un tipo/catalog item explicito para `local` o `custom`.
- **La validacion de skins es demasiado debil y el almacenamiento elegido es fragil.** Impacto: cualquier PNG se acepta y se persiste completo como data URL en `localStorage`; eso puede generar problemas de cuota, rendimiento o estados corruptos si el usuario carga una imagen grande o invalida para Minecraft. Evidencia: `src/App.svelte:279-296`, `src/lib/storage.ts:95-100`. Correccion: validar tamaño/dimensiones, limitar peso y manejar errores de `localStorage` con fallback claro.
- **El backend de dependencias repite demasiada logica de deteccion de distro y planes de instalacion.** Impacto: la misma familia de decisiones esta duplicada para Java y graficos, lo que encarece cambios y facilita drift entre mensajes, comandos sugeridos y auto-install real. Evidencia: `src-tauri/src/commands.rs:105-124,146-259,275-348,364-389`. Correccion: extraer una capa comun de deteccion/plans por plataforma y reutilizarla por dependencia.
- **La definicion de comandos/permisos esta partida entre `build.rs`, permisos manuales y `lib.rs`.** Impacto: hoy funciona, pero la lista de comandos declarada para build solo incluye 4 entradas mientras `invoke_handler` y las capabilities exponen muchas mas; eso aumenta el riesgo de drift y mantenimiento confuso. Evidencia: `src-tauri/build.rs:3-7`, `src-tauri/src/lib.rs:17-30,35-48`, `src-tauri/permissions/*.toml`. Correccion: unificar la fuente de verdad o documentar y automatizar claramente que parte es generada y que parte es manual.
- **Hay codigo muerto y arrastre de iteraciones anteriores en estilos, tipos y copy.** Impacto: aumenta ruido, confunde al mantener y sugiere una UI que ya no existe. Evidencia: estilos no usados para `.optifine-card`, `.optifine-grid`, `.optifine-section` y `.version-item` en `src/app.css:700-860,1069-1096`; tipos de cuenta no usados en `src/lib/types.ts:10-26`; claves de i18n que ya no aparecen en UI en `src/lib/i18n.ts:62-76,97,188-202,223`. Correccion: eliminar lo obsoleto o reintroducirlo con criterio, pero no dejarlo a medio camino.
- **Los previews 3D no degradan bien frente a errores.** Impacto: `CatScene` solo hace `console.error` si falla el modelo y el usuario ve un panel vacio sin contexto ni recovery. Evidencia: `src/lib/CatScene.svelte:199-219`. Correccion: exponer estado de error visible o fallback visual sencillo.

## Low
- **Hay archivos generados o sobrantes que agregan ruido al repo.** Impacto: ensucian la navegacion y vuelven menos claro que forma parte del codigo fuente real. Evidencia: sourcemaps committeados sin `.js` asociados (`vite.config.js.map`, `src/main.js.map`, `src/lib/storage.js.map`, `src/lib/tauri.js.map`, `src/lib/types.js.map`) y assets no referenciados como `hatecreeper.png`, `2026_04_22_spring-trap-24010173.png` y `minecraft_player_model/` segun la busqueda de imports actual. Correccion: borrar lo no usado y evitar commitear artefactos generados.
- **`DESIGN.md` no describe el producto real.** Impacto: hoy documenta un sistema "inspired by Warp" con Matter/warm parchment, mientras la app implementa una estetica pixel/rojo/negro con `Press Start 2P`; el documento puede inducir cambios equivocados. Evidencia: `DESIGN.md:1-16,56-90` frente a `src/app.css:5-27,317,345,813`. Correccion: actualizarlo al estado real o eliminarlo hasta tener una guia vigente.

## Quick Wins
- Desacoplar bootstrap local de cualquier llamada remota y permitir que el launcher abra y liste versiones sin red.
- Rediseñar la resolucion de Java alrededor de la version seleccionada y no de una recomendacion global fija.
- Partir `App.svelte` en stores + componentes de `bootstrap`, `dependencies`, `catalog`, `player-controls` y `logs`.
- Limpiar CSS muerto, tipos no usados, claves i18n huérfanas, sourcemaps y assets sin referencia.
- Endurecer seguridad basica del webview: CSP explicita, fuentes locales y menos HTML inyectado.

## Pendiente Por Sandbox
- No pude verificar `npm run build` porque Vite sigue fallando en este entorno con `spawn EPERM` antes de resolver el proyecto.
- No pude borrar tres binarios sobrantes (`hatecreeper.png`, `2026_04_22_spring-trap-24010173.png` y `minecraft_player_model/{scene.gltf,scene.bin}`) porque la politica del sandbox bloquea los borrados por shell y `apply_patch` no puede operar sobre esos archivos binarios.
